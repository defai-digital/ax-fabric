# Contributing

AX Fabric accepts public issue reports, bug reports, and product feedback.

At this time, AX Fabric does **not** accept unsolicited public code contributions, patches, or pull requests unless they were explicitly requested in advance by the maintainers.

## What We Welcome

- bug reports
- reproducible issue reports
- performance findings
- documentation corrections reported as issues
- feature requests
- commercial and enterprise feedback

## What We Do Not Accept

- unsolicited pull requests
- unsolicited source patches
- large external code drops
- third-party code submitted without prior written approval

Unsolicited code submissions may be closed without review.

## How To Report Issues

When opening an issue, include:

1. operating system and runtime versions
2. package or command affected
3. exact reproduction steps
4. expected behavior
5. actual behavior
6. logs, stack traces, or screenshots when relevant
7. whether the issue affects OSS, Business, or Enterprise usage

## Security Reports

Do not report security-sensitive issues publicly.

Send security reports and disclosure coordination requests to:

- `enquiry@automatosx.com`

Include enough detail for reproduction and impact assessment.

## Licensing and Support

- Open-source use is governed by the project licensing documents in the repository root.
- Business and Enterprise support or licensing enquiries should be sent to `enquiry@automatosx.com`.

## Maintainer Discretion

The maintainers may invite specific external contributors to collaborate on targeted changes. Unless that invitation is explicit, public issue reporting is the supported contribution path.
